CRITICAL BUG FIX: Bearer Token Authentication Vulnerability
===========================================================

SEVERITY: CRITICAL
ISSUE: All protected endpoints were accepting empty tokens due to:
       `token: str = Depends(lambda: "")` pattern

This anti-pattern bypassed authentication entirely. Any client could:
1. Omit the Authorization header
2. Send an invalid Authorization header
3. Send an empty token value
...and still pass authentication checks.

IMPACT: All authenticated endpoints were vulnerable to unauthorized access.

SOLUTION IMPLEMENTED:
====================

1. Created proper Bearer token extraction function
   File: app/core/security.py
   Function: extract_bearer_token(authorization: Optional[str]) -> str
   
   Features:
   - Validates Authorization header presence
   - Enforces "Bearer <token>" format (case-insensitive)
   - Returns proper 401 Unauthorized with WWW-Authenticate header
   - Raises HTTPException on invalid format
   
   Testing: 5/5 unit tests pass

2. Updated 8 endpoint files with proper authentication
   - app/api/v1/endpoints/auth.py (1 change)
   - app/api/v1/endpoints/profile.py (1 change)
   - app/api/v1/endpoints/brand_discovery.py (1 change)
   - app/api/v1/endpoints/cards.py (1 change)
   - app/api/v1/endpoints/dresser.py (1 change)
   - app/api/v1/endpoints/wallpaper.py (1 change)
   - app/api/v1/endpoints/referral.py (2 changes)
   - app/api/v1/endpoints/onboarding.py (1 change)
   
   Total: 10 get_current_user functions + 1 direct endpoint fix = 11 updates

3. Added JWT Secret Key validation
   File: app/core/config.py
   
   Features:
   - Validates secret length (32+ chars in production)
   - Prevents default/weak secrets in production
   - Allows flexible configuration in development
   - Proper error messages for configuration violations

4. Exported new security function
   File: app/core/__init__.py
   - Added extract_bearer_token to imports and __all__
