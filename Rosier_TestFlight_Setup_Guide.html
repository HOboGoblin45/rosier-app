Rosier TestFlight Setup Guide

Deploy to TestFlight from your Windows PC using GitHub Actions. Total time: ~20 minutes.

1 Install OpenSSL on Windows ~3 min

You need OpenSSL to create a signing certificate. Open PowerShell as Administrator and run:

winget install ShiningLight.OpenSSL

After install, close and reopen PowerShell. Verify it works:

openssl version
If winget doesn't work, download from slproweb.com (Win64 OpenSSL Light).

2 Create a Distribution Certificate ~5 min

Step 2a: Generate a private key and Certificate Signing Request in PowerShell:

mkdir C:\Users\ccres\RosierCerts
cd C:\Users\ccres\RosierCerts

openssl genrsa -out rosier_private.key 2048

openssl req -new -key rosier_private.key -out CertificateSigningRequest.certSigningRequest -subj "/CN=Charles Cresci/O=Rosier/C=US"

Step 2b: Go to Apple Developer → Certificates in your browser:

  • Click "+" to create a new certificate
  • Select "Apple Distribution"
  • Click Continue
  • Upload the CertificateSigningRequest.certSigningRequest file from C:\Users\ccres\RosierCerts\
  • Click Continue, then Download the .cer file
  • Save it to C:\Users\ccres\RosierCerts\

Step 2c: Convert to P12 format in PowerShell:

cd C:\Users\ccres\RosierCerts

openssl x509 -inform DER -in distribution.cer -out distribution.pem

openssl pkcs12 -export -inkey rosier_private.key -in distribution.pem -out rosier_distribution.p12 -name "Rosier Distribution"
When prompted for an export password, choose something memorable. You'll need it as a GitHub secret.

3 Create App ID & Provisioning Profile ~3 min

Go to Apple Developer → Identifiers:

  • If com.rosier.app doesn't exist yet, click "+", select "App IDs", choose "App"
  • Description: Rosier, Bundle ID: com.rosier.app (Explicit)
  • Enable capabilities: Push Notifications, Sign In with Apple

Then go to Apple Developer → Profiles:

  • Click "+"
  • Select "App Store Connect" (under Distribution)
  • Select App ID: com.rosier.app
  • Select the Distribution Certificate you just created
  • Name it: Rosier App Store
  • Download the .mobileprovision file to C:\Users\ccres\RosierCerts\

4 Create App Store Connect API Key ~2 min

Go to App Store Connect → Users and Access → Integrations → App Store Connect API:

  • Click "Generate API Key" (or "+" if you have keys already)
  • Name: Rosier CI/CD
  • Access: Admin
  • Click Generate

Copy and save these three values:

  • Issuer ID (shown at the top of the keys page)
  • Key ID (shown next to your key name)
  • Download the .p8 file (can only download ONCE — save it to C:\Users\ccres\RosierCerts\)
The .p8 private key file can only be downloaded once. Do NOT lose it.

5 Base64 Encode & Add GitHub Secrets ~5 min

Run these in PowerShell to encode your certificate and profile:

cd C:\Users\ccres\RosierCerts

# Encode the P12 certificate
$cert = [Convert]::ToBase64String([IO.File]::ReadAllBytes("rosier_distribution.p12"))
Set-Clipboard -Value $cert
Write-Host "Certificate copied to clipboard"

# Paste into GitHub secret, then encode the provisioning profile:
$profile = [Convert]::ToBase64String([IO.File]::ReadAllBytes("Rosier_App_Store.mobileprovision"))
Set-Clipboard -Value $profile
Write-Host "Profile copied to clipboard"

# Read the API key content:
$key = Get-Content "AuthKey_XXXXXXXXXX.p8" -Raw
Set-Clipboard -Value $key
Write-Host "API key copied to clipboard"

Go to GitHub → Settings → Secrets → Actions and add these 7 secrets:

Secret NameValue
APPLE_DEVELOPER_ID52SKBHZK3L
IOS_CERTIFICATE_BASE64Paste the P12 base64 string
IOS_CERT_PASSWORDThe password you set when exporting P12
IOS_PROVISIONING_PROFILE_BASE64Paste the provisioning profile base64 string
APP_STORE_CONNECT_KEY_IDThe Key ID from App Store Connect
APP_STORE_CONNECT_ISSUER_IDThe Issuer ID from App Store Connect
APP_STORE_CONNECT_PRIVATE_KEYFull contents of the .p8 file (including BEGIN/END lines)

6 Create App in App Store Connect ~2 min

Go to App Store Connect → My Apps:

  • Click "+" → "New App"
  • Platform: iOS
  • Name: Rosier
  • Primary Language: English (U.S.)
  • Bundle ID: com.rosier.app
  • SKU: rosier-app-001
  • Full Access: your user

7 Deploy to TestFlight

Once all secrets are set, trigger a TestFlight build by creating a version tag. You can do this from the GitHub web UI or PowerShell:

# Option A: From GitHub.com
# Go to your repo → Releases → "Create a new release"
# Tag: v1.0.0, Target: main, Title: "Rosier v1.0.0"
# Click "Publish release"

# Option B: From PowerShell (if you have git installed)
git tag v1.0.0
git push origin v1.0.0

This automatically triggers the deploy-testflight job in GitHub Actions, which builds the app on a macOS runner, signs it with your certificate, and uploads it to TestFlight.

Monitor progress at: GitHub Actions

8 Test on Your iPhone

  • Download TestFlight from the App Store on your iPhone
  • Sign in with the same Apple ID used for App Store Connect
  • After the build processes (~10-15 min after upload), Rosier will appear in TestFlight
  • Tap Install to test the app on your device
First-time TestFlight builds may take up to 30 minutes for Apple to process before they appear.

Checklist

  • OpenSSL installed on Windows
  • Distribution certificate created and converted to P12
  • App ID (com.rosier.app) registered
  • Provisioning profile downloaded
  • App Store Connect API key created and .p8 saved
  • All 7 GitHub secrets added
  • App created in App Store Connect
  • Version tag pushed to trigger build
  • TestFlight installed on iPhone
