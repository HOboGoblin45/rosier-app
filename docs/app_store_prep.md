## 7. Common Rejection Reasons & Solutions

### Issue: Functionality Issues

**Rejection Reason:** "This app demonstrates bugs that interfere with its intended functionality."

**Prevention:**
- Test thoroughly on real devices (not just simulator)
- Test on older iOS versions and devices
- Test on slow networks (3G/4G, not just Wi-Fi)
- Verify affiliate links work correctly
- Test account creation and login flows

**Solution if Rejected:**
- Reproduce the bug and fix it
- Add detailed testing notes in resubmission
- Provide updated build with fixes
- Include updated release notes explaining changes

---

### Issue: Misleading Metadata

**Rejection Reason:** "Your app description promises features not present in the app."

**Prevention:**
- Ensure description matches actual functionality
- Don't promise features in development
- Don't claim integrations not yet implemented
- Screenshots must match app's actual UI
- Keywords must be relevant to actual app content

**Solution if Rejected:**
- Update description to match actual functionality
- Replace misleading screenshots
- Remove unavailable features from description
- Resubmit with updated metadata

---

### Issue: Privacy & Tracking Concerns

**Rejection Reason:** "Your app collects user data but does not sufficiently disclose this in the Privacy Policy."

**Prevention:**
- Comprehensive Privacy Policy addressing GDPR/CCPA
- Clear disclosure of data collection in app
- Proper handling of location and tracking permissions
- Privacy Nutrition Label accurately filled out
- No hidden data collection

**Solution if Rejected:**
- Update Privacy Policy with complete disclosures
- Verify all permissions have user consent
- Update Privacy Nutrition Label if needed
- Add in-app privacy disclosures
- Resubmit with clarified privacy practices

---

### Issue: Affiliate Link Disclosure

**Rejection Reason:** "App does not adequately disclose affiliate relationship."

**Prevention:**
- Include clear affiliate disclosure in Terms of Service
- Disclose commission earning model in app
- Include FTC disclosure about earnings
- Make disclosure prominent and accessible
- Explain that commission doesn't affect pricing

**Solution if Rejected:**
- Add prominent affiliate disclosure in app
- Update Terms of Service to emphasize affiliate model
- Create in-app disclosure when clicking affiliate links
- Add to App Store description if not already present
- Resubmit with enhanced disclosures

---

### Issue: Metadata Issues

**Rejection Reason:** "The Keywords you provided do not match the app content."

**Prevention:**
- Use keywords directly related to actual features
- Don't keyword stuff or use misleading keywords
- Research competitor keywords for reference
- Include long-tail keywords for niche features
- Test keywords with actual app functionality

**Solution if Rejected:**
- Replace keywords with accurate ones
- Ensure keywords reflect actual app features
- Use analytics to find effective keywords
- Research App Store search trends
- Resubmit with revised keywords
